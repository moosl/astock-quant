## 2. 关键设计决策

### 2.1 复用纪律（copy-then-modify，对应 §3.1）

完全不动的文件清单（mtime 应停在 Stage 2 之前）：
- `data/` 全部 4 个文件
- `factors/` 全部 7 个文件
- `models/base.py` / `models/splits.py` / `models/direction.py`
- `backtest/constraints.py` / `portfolio.py` / `engine.py` / `metrics.py`
- `contracts.py`（`Prediction.value` 是 float 已支持 ②，`target_type="return"` 是 Literal 已枚举）
- `config/settings.py`

只动 lead 限定的 4 个文件 + 新增 4 个测试。

### 2.2 `return_label` 与 `direction_label` 的数学恒等

`return_label = pct_change(horizon).shift(-horizon)`，**不二值化**直接返回 float。
`direction_label = (return_label > threshold).astype(float)`。

这是数学恒等式（`test_return_label_consistent_with_direction` 守住）。如果将来谁误改 return_label 的 shift 链 / groupby 写法 / for_training 语义，这条测试会立刻挂。

三道防 look-ahead 全部对齐 direction：
1. 数据层：上游 panel 经 `cache.truncate_by_date` 截断时，未来 horizon 天的 close 不可见
2. 切分层：`models/splits.py` purge_gap >= horizon 守住「训练样本的 N 日未来不能伸进验证集」
3. 训练前：`align_xy(drop_label_nan=True)` 把 NaN 样本扔掉

### 2.3 `ReturnRegressor` 的持久化路径

完全复用 DirectionModel H1 修复后的**全公开 API 模式**：
- `Booster.save_model(path)` + sidecar `<path>.feature_names.json`
- `lgb.Booster(model_file=path)` 加载，load 后 `self._reg = None` 彻底脱钩 sklearn wrapper
- predict 路径走 `booster.predict(X)`，不依赖 `LGBMRegressor` 的私有 `_Booster / _n_features / _classes`

测试 `test_load_does_not_touch_private_attrs` 断言这条不变量，与 DirectionModel H1 同款命门。

### 2.4 回归契约（区别 direction 的核心）

| 字段 | direction | return |
|---|---|---|
| `Prediction.value` | 硬分类 0.0 / 1.0 | 连续 float 收益率 |
| `Prediction.score` | P(涨) ∈ [0, 1] | = value（连续可比，下游按阈值过滤） |
| `Prediction.proba` | (P(跌), P(涨)) | **None**（回归无概率） |
| `target_type` | `"direction"` | `"return"` |
| LightGBM | `LGBMClassifier(objective="binary")` | `LGBMRegressor(objective="regression")` |
| 训练 eval_metric | `["binary_logloss", "auc"]` | `["rmse", "l1"]` |
| 评估指标 | Acc / AUC / log_loss | **RMSE / MAE / R² / IC / rank-IC** |

回归契约（`score == value` 且 `proba is None`）由 `test_prediction_value_equals_score_proba_none` 守住。

### 2.5 `SignalGenerator` 的 return 分支升级

Stage 1 stub 是硬编码 `v > 0.005` / `v < -0.005`。P9 升级为可配阈值：

```python
SignalGenerator(
    return_buy_threshold=+0.02,   # 默认 +2%
    return_sell_threshold=-0.02,  # 默认 -2%
)
```

阈值默认挑选理由：
- 模型 horizon=5 日
- A股 单日波动 1-2% 是常态
- 5 日累计 ±2% 是「值得动手」的最低门槛 —— 低于此值不如不动（成本会吃掉收益）

强度公式（buy 时）：
```
strength = min(1, (value - buy_thr) / |buy_thr|)
```
- value == buy_thr → strength = 0（刚达阈）
- value == 2×buy_thr → strength = 1（封顶）

阈值校验：`return_buy_threshold < return_sell_threshold` 抛 ValueError（与 direction 阈值校验同款）。

## 3. 实测结果（诚信的弱基线）

### 3.1 默认阈值（signal ±2% / backtest direction-style 0.55/0.45）

```
=== 训练 / 评估指标（② 收益率回归）===
  train_size = 25283         ← 与 P3b 一致（共用 splits）
  valid_size = 5780          ← 与 P3b 一致
  train_end  = 2025-06-30
  valid_start = 2025-07-15
  y_train_mean = -0.000363   ← 5 日累计收益均值接近 0（合理：训练区间整体平稳）
  y_train_std  = 0.048076    ← ~4.8% 标准差（5 日累计波动）
  y_valid_mean = 0.000930
  y_valid_std  = 0.043888
  rmse = 0.043926            ← 接近 y_valid_std 0.0439 → 模型几乎等同「猜均值」
  mae  = 0.027869
  r2   = -0.001906           ← 轻微负值，模型预测略差于「常数均值」
  ic   = -0.042557           ← 轻微负相关（Pearson IC）
  rank_ic = +0.002740        ← 基本 0（横截面排序无 alpha）
```

**回测**：默认 backtest_threshold=0.55（direction-style）→ 0 笔交易 / 0% 收益（return scale 永远达不到 0.55）。

### 3.2 信号阈值放宽（±0.5% / ±0）

```
signal ±0.5%：5780 全 hold → 0 交易
signal ±0：5780 = 433 buy + 5347 sell（绝大多数预测负收益）
但回测仍 0 交易（buy_thr=0.55 不变，return scale 错位）
```

### 3.3 回测阈值匹配 return scale（buy_thr=0, sell_thr=-0.005）

```
=== 回测指标 ===
  n_trades = 5
  n_buy_orders = 5, n_sell_orders = 0
  total_return = -0.1733     ← -17.33%
  sharpe = -1.169
  sortino = -1.335
  max_drawdown = -0.2152     ← -21.52%
```

诚信解读（与 Stage 1 ① direction「14 笔 / +5.92% 不是 alpha」一脉相承）：

- **IC=-0.04 + R²<0 → 模型基本随机**：预测的收益率与真实收益率相关性微弱（甚至轻微负相关），rank-IC ≈ 0 说明横截面排序也无能力
- **回测亏损正常**：模型选股能力 ≈ 0，5 笔交易 + 交易成本（佣金 + 印花 + 滑点）拖累 → 亏损 17% 完全是预期
- **比 ① direction 还弱**：① AUC=0.5131 略高于 0.5（极弱信号），② IC=-0.04 略低于 0（接近随机但方向反），都是「学习用诚信弱基线」的典型表现

### 3.4 阈值 mismatch 提醒（pipeline 用法注意）

`BacktestRunConfig.buy_threshold / sell_threshold` 默认 0.55 / 0.45 是 **direction-style**（概率域）。
回归任务下应**显式覆盖**：

```python
from astock_quant.backtest.engine import BacktestRunConfig
from astock_quant.pipeline.run_return import run_return

cfg = BacktestRunConfig(
    buy_threshold=0.0,        # ← 预测收益率 > 0 就买
    sell_threshold=-0.005,    # ← 预测收益率 < -0.5% 就卖
)
r = run_return(
    backtest_config=cfg,
    signal_buy_threshold=0.0,
    signal_sell_threshold=-0.005,
)
```

这是「不动 engine、不增 selection_mode 参数」的复用代价（与设计 §2.3 一致）。Stage 4 可考虑：让 BacktestRunConfig 按 target_type 自动选合理默认（属于设计层面优化，不在 P9 scope）。

## 5. 待 ③ ④ 处理的事项

### 5.1 ③ ranking 阶段（P10）即将面对的命门

`return_label` 的 `pct_change.shift` 链是 ranking_label 的上游：

```python
# 设计 §5.1 横截面 rank look-ahead 高风险
def ranking_label(...):
    future_ret = return_label(panel, horizon=horizon)  # step 1：用 return_label
    rank = future_ret.groupby(level="date").rank(pct=True)  # step 2：必须按日期 group
```

**绝对不能**对全 panel `future_ret.rank()`（会用全样本分布算分位）。
设计 §5.4 group-aware split 命门测试要在 P10 启动前补上。

### 5.3 回测阈值默认值（设计层债务）
