## 二、② 收益率回归是什么——硬币正反 vs 骰子点数

### 2.1 跟 ① 涨跌方向的区别

Stage 1 一路做的是 ① 涨跌方向，**二分类**。这次 P9 做的是 ② 收益率回归，**回归**。两个词都是机器学习术语，得解释清楚。

> **名词解释·二分类（Classification）**：让模型从两个选项里选一个。比如「明天涨」or「明天跌」——非黑即白，只有两个答案。
>
> **名词解释·回归（Regression）**：让模型预测一个**具体的数字**。比如「明天涨 0.5%」「明天跌 1.2%」——连续值，可以是 -100% 到 +100% 任意一个数。

最直观的类比：

> **二分类（① direction）= 猜硬币正反**：你只用回答「正面」或「反面」，没有第三个选项。
>
> **回归（② return）= 猜骰子点数**：你要给出一个具体数字，骰子掷出来 1-6 你猜的越接近越好。猜「3 点」可能差 0 也可能差 5（如果实际是 8 点的骰子）。
>
> 区别：
> - **硬币题**：错就是错，对就是对。算 Accuracy（猜对率）。
> - **骰子题**：你说「3 点」但实际是「3.5 点」也算挺接近的；如果你说「5 点」实际是「3 点」差得就多了。**算"差多少"，不是算"对错"**。

### 2.2 为什么要做 ②

① 已经能答「涨还是跌」——为什么还要再做一个 ② 答「涨多少」？

因为**「涨多少」比「涨」信息量大**：

- 如果模型预测「明天涨 0.1%」——你大概会**不动手**（涨这么点连交易成本都不够）
- 如果模型预测「明天涨 5%」——你可能**会去买**

只看 ① 你做不出这种区分。② 给你的是"涨跌幅度"，让下游策略可以**按幅度大小决定动手力度**。

报告 01 当时讲「4 类预测目标共用前半条流水线」就是想覆盖这种渐进的复杂度——从最简单的「猜涨跌」一路扩到「预测排序」「直接出操作建议」。

## 四、关键设计：return_label 与 direction_label 的数学恒等

这一节讲的是 P9 最讲究的一处工程细节。技术词难懂，但用类比讲清楚很值得。

### 4.1 两个标签必须满足的数学关系

> **名词解释·标签（label）**：报告 03 讲过——机器学习里给训练数据贴的「标准答案」。
>
> ② return 的标签：未来 5 天涨跌幅（一个数字，比如 +0.03 表示涨 3%）
> ① direction 的标签：未来 5 天涨没涨（0 或 1，涨就是 1 跌就是 0）

这两个标签**必须满足一条数学等式**：

```
direction_label = (return_label > 0).astype(float)
```

意思是：**direction 标签就是 return 标签「大于 0 就标 1，否则标 0」**。

这听起来理所当然——但**代码层面有可能漂移**。比如某次 refactor 时谁不小心改了 return_label 的"未来位移"逻辑（比如把 shift(-5) 改成 shift(-6)），但忘了同步改 direction_label——**两个标签内部对不上了**，模型用的"未来 5 天涨跌"跟"未来 6 天涨幅大于 0"对应起来，无声无息地坏掉。

### 4.2 用类比讲讲为什么这条数学等式重要

> **类比**：你开两家餐厅，一家卖**牛排定食**（return_label），一家卖**牛排沙拉**（direction_label）。
>
> 两家共用一个厨房、共用一个采购员、共用一份牛排原料。
>
> 有一天采购员换了进货标准（比如「200g/份」改成「180g/份」），他改了牛排定食那边的菜单，**但忘了同步给牛排沙拉那边**。
>
> 结果牛排沙拉那边按 200g 卖给客人，但厨房只给 180g——**客人投诉了**。
>
> 这种"两家店共用同一原料但配方对不上"的 bug，**靠人脑盯不住**。需要一个自动监督机制。

### 4.3 命门测试守住这条等式

P9 写了一个测试叫 `test_return_label_consistent_with_direction`，里面就一行核心断言：

```python
derived_dir = (y_ret > 0).astype(float)     # 用 return 标签推算出 direction
pd.testing.assert_series_equal(derived_dir, y_dir)  # 必须跟真正的 direction 完全一致
```

意思是：**每次跑 CI，自动测试都会用 return_label 推算一遍 direction_label，跟真的 direction_label 比对**。一旦哪个标签漂移了，测试立刻报警。

> **名词解释·命门测试**：报告 03 / 05 / 07 都讲过——专门盯着"这件事一旦破了，整个项目就坏"的某条铁律。这种测试是"长期值守的质检员"。

P9 一共加了 **3 个命门测试**，刚才讲的是第 1 个。下一节讲另外 2 个。

### 5.2 命门 3：pipeline 真接的是 return_label，不是 direction_label

> **名词解释·wiring（接线）**：报告 07 讲过——pipeline 里各步骤之间「谁调谁」的连线。报告 07 那次 P7 wiring 5 处接线缺失（DeepSeek 一次没被调用）的教训记忆犹新。

| MAE（绝对误差）| **0.028** | 平均预测误差 2.8 个百分点（对 5 日涨幅而言挺大）|
| **R²** | **-0.0019** | **负数**——模型比"直接猜常数均值"还略差 |

| 回测最大回撤 | -21.52% | 最惨那次跌 21.5% |

### 6.2 R² = -0.0019 是什么意思

R² 这个指标比较抽象，用大白话讲一下：

> **名词解释·R²（决定系数）**：衡量回归模型"比 baseline 强多少"的指标。范围一般 0~1：
> - **R² = 1**：完美预测
> - **R² = 0**：模型预测能力 = "直接猜常数均值"（baseline）
> - **R² < 0**：模型**比 baseline 还差** ——意味着「不如直接预测一个常数 0」

我们这次 R² = **-0.0019**——**模型预测涨跌幅，还不如直接猜「涨 0」**。

类比：

> 你雇了个预测家给你猜每天涨多少。**你也可以选不雇他、直接每天都猜「涨 0」**。
> 一年下来，**这哥们的成绩比你直接猜 0 还差一点点**。
> 那你雇他干嘛？

这就是 R² 负数的含义。

### 6.3 跟 ① / Stage 2 一脉相承——这套系统"写得对但模型不会赚钱"

回顾一下我们 3 次跑模型的诚信结果：

| 阶段 | 任务 | 关键指标 | 诚信解读 |
|---|---|---|---|
| Stage 1 (① direction) | 二分类 | **AUC = 0.5131** | 略高于随机 0.5，几乎是抛硬币 |
| Stage 2 (LLM 因子) | 加情绪因子 | **importance = 0.0** | 模型完全不看 LLM 列（数据太稀）|
| **Stage 3 (② return) 本期** | **回归** | **R² = -0.0019 / IC = -0.04** | **比常数均值还差一点点，几乎随机** |

**3 次结果一脉相承**：

> **系统写得对（代码 + 测试 + look-ahead 防线全过），但模型不会赚钱**。

这是"学习/研究"项目的诚实状态。**不是 bug，是 A股 短期方向真的接近随机过程**——大盘蓝筹股的 5 日涨跌，本来就在统计意义上接近抛硬币。

报告 05 讲过：「学术界经验天花板就在 0.55-0.58（AUC）」。我们没做任何提升技巧（横截面归一化、行业中性化、超参数调优、多模型集成）——纯裸跑 baseline，**这些数字就是 baseline 该有的样子**。

### 6.4 「-17.33% 亏损」是怎么回事

5 笔交易亏了 17.33%——听起来挺糟。**但要把它放到正确的语境里**：

1. **模型预测能力 ≈ 0**：IC = -0.04 说明预测和真实**几乎不相关**（甚至轻微反向），模型选的 5 只票本质上是随机选的
2. **5 笔交易 + 交易成本**：报告 04 讲过——每笔交易要交手续费、印花税、滑点（合起来叫"过路费"），5 笔交易被收 5 次过路费
3. **如果模型真有 alpha 应该赚钱**：但 IC = -0.04 说明它没 alpha，所以**亏损是必然的**——不亏才奇怪

跟报告 04 三组对照那次「随机 score 瞎猜亏 10.6%」一模一样的逻辑——**模型没本事时，做交易就是给市场送过路费**。

### 6.5 与 ① direction 的并排诚信对比

| 维度 | ① direction (P3b) | ② return (P9) |
|---|---|---|
| 评估指标 | AUC=0.5131（略高于随机 0.5）| R²=-0.0019 + IC=-0.04（接近随机但方向略反）|
| 默认回测 | 14 trades / +5.92% / Sharpe 1.10 | 5 trades / -17.33% / Sharpe -1.17 |
| 诚信解读 | "不是 alpha，是少交易 + 市场 β"（报告 04）| "比常数均值差一点点，5 笔被成本拖死"（本报告）|

## 八、接下来：P10 ③ 横截面 Top N + P11 ④ 信号扩展

P9 完成 ②，Stage 3 还剩 2 个目标：

```
② return (P9 ✅) ──→ [③ ranking (P10)] ──→ [④ trade_signal (P11)] ──→ Stage 3 收尾
                      ▲▲▲▲ 下一站
```

### 8.1 ③ ranking（P10）干什么

> **名词解释·横截面 Top N**：给一篮子股票按预测打分**排个名**，挑前 N 名买入。
>
> 跟 ① / ② 的区别：
> - ① / ②：**每只票独立判断**「这只票本身好不好」
> - ③：**横向比较**「这 30 只票里，哪几只最好」
>
> 类比：① / ② 像给学生**单独打分**（这学生 80 分还是 60 分）；③ 像给全班学生**排名**（30 个人里挑前 5 名）。

P9 文档已经给 P10 留了一个**重要预警**——③ ranking 的标签计算里有一处**横截面 look-ahead 高风险**：千万不能对全 panel 一起 `.rank()`（会用全样本分布算排名），必须**按日期 groupby 后再 rank**。

> **名词解释·横截面 look-ahead**：「同一天的 30 只票之间」的排名，**不能用未来某天的票一起算**。否则等于知道未来。
>
> 类比：今天给班里同学排名，**不能用下周的成绩参与排名**。

这个命门测试要在 P10 启动前补上——auditor 也明确点了这一条。

### 8.2 ④ trade_signal（P11）干什么

> **名词解释·trade_signal（买卖信号）**：模型直接输出「现在该买 / 该卖 / 该持有」，不只是给个分数让你自己定阈值。

这块比 ②③ 复杂——需要"路径标注"（不只是看终点涨没涨，还要看路径中有没有触及止盈止损线），需要 panel 含 `high / low` 列（已确认有），引擎要补 stop-loss / take-profit 触发逻辑。这块还在设计阶段。
